## 二、核心用户旅程（5 条主要路径）

### 旅程 A：批注 → 生成记忆卡（产品核心流程）

```
点"批注演示＋"选场景
   → 在模拟页面上画批注（铅笔/荧光笔/橡皮，撤销重做）
   → 点"批注完成"
   → 截图合成 + 压缩
   → 【AI 第1步】豆包视觉模型看图，提取结构化要点
   → 【AI 第2步】DeepSeek 把要点精修成详情页正文（流式）
   → 截图飞入左下角 + Toast"已同步至AI记忆"
   → 新卡片出现在 Feed 流
```

### 旅程 B：上传图片 / 导入资料 → 生成记忆卡

```
侧边栏点上传图标，选图片            侧边栏/弹窗导入文件(图/PDF/文本)
   → 走和旅程A几乎一样的流程              → 走 /api/import 代理
   (全图分析，hasAnnotations=false)        (图片走豆包，文本/链接走 DeepSeek)
```

### 旅程 C：浏览 / 复习

```
左侧选学科 或 "全部"
   → Feed 按日期看卡片 + 每日智能总结（可点赞/踩/重新生成）
   → "全部"视图可按知识类型标签 + 高频关键词标签筛选
   → 点卡片 → 右侧详情抽屉看深度解析
   → 点正文里的蓝色名词 → 下钻看该概念的二级讲解页
```

### 旅程 D：基于卡片做练习

```
进入"习题"Tab / 选中若干卡片 → 打开生成抽屉
   → 选模块（习题 / 思维导图 / 难点讲解 / 交互演示）和难度
   → AI 生成
   → 出题：先规划"组卷蓝图"，再据蓝图逐题出（两步）
   → 结果存入"学习记录"
```

### 旅程 E：交互演示（最复杂的一条 AI 链）

```
选中卡片 → 选"交互演示"
   → 【AI 第1步】DeepSeek 设计"交互方案"(plan)：交互什么、几个可调参数、画面长什么样
   → 【AI 第2步】DeepSeek 根据 plan 生成一个可运行的 React 单文件组件
   → 在 Sandpack 沙箱里实时运行，学生可拖滑块调参数看变化
```

## 四、数据与状态（不懂代码也要知道的几件事）

| 问题 | 答案 |
|------|------|
| 数据存在哪？ | 浏览器本地 **IndexedDB**（封装在 `utils/db.ts`，通过 `hooks/useMemoryDB.ts` 读写） |
| 有云端数据库吗？ | **没有**。换浏览器、清缓存、换设备，数据就没了。Demo 阶段够用 |
| 内置的演示卡片从哪来？ | `data/initialData.ts` + `data/builtin*.ts` + `data/*Seed*.ts`，首次打开自动灌入 |
| 新生成的卡片会持久化吗？ | 会（写入 IndexedDB），但只在当前浏览器 |
| 首屏会不会卡死在 loading？ | 有 8 秒"看门狗"兜底（`useMemoryDB.ts`），超时就用内存数据放行 |
| 数据结构定义在哪？ | `src/types.ts`（卡片 `CardData`、习题 `QuizQuestion`、交互 `InteractionPlan` 等都在这） |

## 五、目录结构速览

```
项目根/
├── api/                      # ⭐ 后端代理（Vercel Edge 函数）+ 部分提示词
│   ├── doubao.ts             #   豆包视觉代理（批注/上传出卡）含视觉提示词
│   ├── deepseek.ts           #   DeepSeek 文本代理（流式/非流式通用通道）
│   └── import.ts             #   资料导入代理（图片走豆包 / 文本链接走 DeepSeek）
├── src/
│   ├── prompts/              # ⭐ 提示词大本营（除视觉外的大部分提示词）
│   │   ├── index.ts          #   详情页/概念下钻/每日总结/卡片自测 提示词
│   │   ├── exercise.ts       #   习题/导图/难点/交互 提示词
│   │   └── interactiveFewShots.ts  # 交互代码的"金标准"示例
│   ├── utils/
│   │   ├── api.ts            #   前端调模型的统一入口（streamText/callText/callDoubao）
│   │   ├── json.ts           #   LLM JSON 容错解析（很重要，见 03/04 文档）
│   │   └── interactiveGeneration.ts  # 交互演示两步编排
│   ├── features/             #   各功能模块（见上面对照表）
│   ├── app/App.tsx           # ⭐ 主装配 + 批注出卡编排
│   ├── data/                 #   内置演示数据
│   ├── hooks/useMemoryDB.ts  #   本地数据库读写
│   └── types.ts              #   所有数据结构定义
├── docs/handover/            #   你正在读的这套交接文档
└── .env.local               #   本地 API Key（不进 git）
```
