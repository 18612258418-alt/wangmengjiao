## 这个项目到底是什么

**AI 记忆**：一款面向大学生的智能学习辅助工具。核心闭环是——

> 学生在任意学习内容（网页 / 课件 / 笔记 / 上传的图片或文件）上圈选批注 → AI 自动分析内容 → 生成结构化的「记忆卡片」并按学科归档 → 学生可以基于卡片继续出习题、看思维导图、玩交互演示，把"看过"变成"记住"。

它是一个**前端 Demo**（React 单页应用），AI 能力通过调用两个大模型实现，部署在 Vercel 上，有一个稳定的内测地址给大学生体验。

## 一个最重要的心智模型（务必先建立）

这个项目里有两类"资产"，价值完全不同：

| 类型 | 是什么 | 谁在维护 | 改起来 |
|------|--------|---------|--------|
| **代码（壳）** | React 组件、UI 布局、动画、状态管理 | 主要是 AI（Cursor）生成 | 容易，可让 AI 重写 |
| **提示词 + 编排（核）** | 给大模型的指令、多步骤怎么串、输出格式怎么约束 | **你**，需要人来设计判断 | 要小心，是产品的真正灵魂 |

**结论**：你每周迭代时，真正在动脑、需要你做设计判断的，**80% 是提示词和编排逻辑**，而不是代码。代码层面交给 Cursor 去实现就好。所以这套文档把最大篇幅给了 [`03_大模型编排地图`](./03_大模型编排地图.md) 和 [`04_提示词设计规范`](./04_提示词设计规范.md)。

## 30 分钟上手路径

按顺序做完这几步，你就能跑起来并理解全貌：

1. **跑起来**（10 min）——按 [`05_运行部署与运维`](./05_运行部署与运维.md) 的"本地运行"小节，用 `vercel dev` 在本地打开应用，亲手批注一次、上传一张图、出一套题，感受完整流程。
2. **看产品为谁服务**（5 min）——读 [`01_产品与用户`](./01_产品与用户.md)，建立"该听用户哪些反馈"的判断力。
3. **看界面地图**（5 min）——读 [`02_架构与用户旅程`](./02_架构与用户旅程.md)，知道每个界面对应哪个文件。
4. **看 AI 怎么编排的**（10 min）——读 [`03_大模型编排地图`](./03_大模型编排地图.md)，这是核心，理解"哪个功能 → 哪个提示词 → 哪个模型 → 几步完成"。

之后再按需查阅 04（怎么改提示词）、06（怎么指挥 Cursor 改东西）。

## 关键事实速查（背下来省事）

| 项 | 值 |
|----|-----|
| 技术栈 | React 18 + Vite 6 + TypeScript + Tailwind CSS 4 |
| 视觉模型 | 豆包 `doubao-vision-pro-32k`（火山引擎 Ark）——负责"看图" |
| 文本模型 | DeepSeek `deepseek-v4-pro` / `deepseek-v4-flash`——负责"写内容" |
| 提示词都在哪 | `src/prompts/index.ts`、`src/prompts/exercise.ts`、`api/doubao.ts`、`api/import.ts` |
| 后端代理在哪 | `api/` 目录（3 个 Vercel Edge 函数：`doubao.ts` / `deepseek.ts` / `import.ts`） |
| 部署平台 | Vercel（push 到 GitHub 即自动部署） |
| 数据存哪 | 浏览器本地 IndexedDB（没有云端数据库，换浏览器/清缓存即清空） |
| 开发分支 | `main`（你日常改这里） |
| 内测分支 | `release/demo-v1`（合并到这里才更新内测用户的地址） |

## 交接当下的"待办与坑"（务必看一眼）

- **API Key 是有成本的真实密钥**，存在 `.env.local`（本地）和 Vercel 后台（线上）。不要泄露、不要提交到 git。详见 05 文档。
- 有**两个 Vercel 项目**（开发版 / 内测版），它们的环境变量要分别维护。
- 历史文档里的环境变量名（`VITE_DOUBAO_API_KEY` 等）已经**作废**，现在用的是不带 `VITE_` 前缀的服务端变量。详见 05 文档的勘误。
- 几个提示词函数（`buildGroupSummaryPrompt`、`buildDailyReportPrompt`）已定义但**当前没有接线使用**，属于历史遗留，想做"每日报告"功能时可复用。
